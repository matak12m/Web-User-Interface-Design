Web Protocols

HTTP

HyperText Transfer Protocol
Hypertext - text with added value, for example, a link to a different website. the Web is built upon this concept.
This protocol helps to redirect the user to the appropriate website based on the URL.

HTTPS

HyperText Transfer Protocol - Secure
this protocol, in addition to HTTP, provides additional security measures to make sure that your information cannot be stolen - passwords, credit card information, etc.
This is usually done by encrypting your information.

TCP/IP

TCP = transmission control protocol. IPs are strings of random numbers unique to local servers/computers - they act as an adress.
TCP helps users redirect to the correct IP. IPs are a string of numbers, however, people are much better at remembering names and words instead of random numbers,
so TPC translates domain names (ex. itcarlow.ie) into these IP numbers.

FTP

File Tranfer Protocol
this protocol handles transferring files - downloading, uploading, etc.
Like its name implies, URLs lead to a single location of a specific resource. The protocol tells it what it is looking for, while the rest of it leads to the right file.
The domain and sub-domain lead to a server, where the file withing a folder is.

Cookies and Browsers

Cookies are used to store data about you for your own convenience, or for marketing purposes.

for example, a Cookie on a weather forecast webpage could hold data on what cities you looked up previously, so you don't have to look for them again, and instead have them appear on a "previously viewed" section.

however, cookies can also harvest data used by advertisers, such as your interests, age, and other personal information, that is used to show you relevant ads.

Browsers and Rendering Engines

different browsers run on different REs, for example, Chrome, OperaGX, and Edge run on Chromium, while Firefox runs on a differnt Rendering Engine.

The concept of caching

Caching refers to storing data client-side, or proxy-side. These are used to save on processing for for the webpage/server.

For example, you could cache some data the user has entered into a form, so that if they accidentally close the page, they can reload and have their data saved.
